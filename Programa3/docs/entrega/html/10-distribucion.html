10 - Distribucion

APK firmado y App Distribution

La distribucion se hizo con Firebase App Distribution. El mismo APK firmado que Melanie instala se conserva como respaldo en la entrega para que el evaluador pueda instalarlo si no usa el canal de Firebase.

Firma

El APK esta firmado con el keystore local de release. La huella SHA-1 de release fue agregada a la restriccion Android de la API key de Google Maps para evitar mapas en blanco en el build distribuido. 

Package: com.example.bachewatch
Release SHA-1:
C3:B3:33:A7:CC:FE:36:41:B3:41:13:D8:7D:9F:18:91:B7:25:54:BF

Comandos usados

cd Programa3
./gradlew clean assembleRelease
cp app/build/outputs/apk/release/app-release.apk docs/entrega/app-release.apk
firebase deploy --only firestore:rules,firestore:indexes,storage --project bachewatch-upiicsa
firebase appdistribution:distribute docs/entrega/app-release.apk \
  --app 1:1008157923999:android:2a6d136350f1b16bcee0aa \
  --project bachewatch-upiicsa \
  --groups testers \
  --release-notes "BacheWatch release candidate"
Nota para Melanie. Si la instalacion queda en "downloading", debe aceptar la invitacion con la misma cuenta Google que usa Google Play Services en el telefono.
