AnimoChat - Implementation Plan

Updated: 2026-06-02
Course: 6NM61 Programacion Movil, UPIICSA
Team: Gonzalez Calzada Maximiliano, Sosa Montoya Melanie Rubi
Source: docs/source-notes-2026-06-02.html

This is the Program 2 planning snapshot. It mirrors the Program 1 convention: domain language lives in docs/CONTEXT.html, decisions live in docs/adr/, work ownership lives in docs/work-division.html, and executable task specs live in docs/tasks/.

1. Planned Scope

AnimoChat is an Android app that lets a student write fears, doubts, or academic worries and receive motivational feedback generated by an AI provider.

The v1 deliverable includes:

  • Jetpack Compose chat screen with student and assistant bubbles.
  • Text input, send action, loading state, retry/error state, and clear chat.
  • HTTP call from Android to an AI provider.
  • JSON response parsing into a local model.
  • Prompt policy that conditions every request with negative cases.
  • Crisis/unsafe redirect for killing, self-harm, suicide, or violence content.
  • Local fallback response when API or JSON parsing fails.
  • Manual test script covering normal motivational and negative cases.

4. Architecture

The classroom-note architecture is preserved:

Android app -> HTTP -> AI API -> JSON -> UI Compose

Implementation shape:

Compose ChatScreen
        |
ChatViewModel
        |
AiChatRepository
        |
GeminiRestClient
        |
HTTP generateContent request
        |
AiFeedback JSON
        |
Compose renders response

The UI depends on AiChatRepository, not on Gemini classes or raw HTTP.

5. Data Model

Planned Kotlin models:

enum class ChatRole {
  Student,
  Assistant
}

data class ChatMessage(
  val id: String,
  val role: ChatRole,
  val text: String,
  val createdAtMillis: Long,
)

enum class FeedbackCategory {
  Motivational,
  AcademicStress,
  CrisisOrUnsafe,
  OutOfScope,
}

data class AiFeedback(
  val category: FeedbackCategory,
  val message: String,
  val nextSteps: List<String>,
  val followUpQuestion: String?,
  val resources: List<SupportResource>,
)

data class SupportResource(
  val label: String,
  val value: String,
)

Expected JSON shape from the provider:

{
  "category": "academic_stress",
  "message": "string",
  "nextSteps": ["string"],
  "followUpQuestion": "string or null",
  "resources": [
    { "label": "Linea de la Vida", "value": "800 911 2000" }
  ]
}

6. Prompt Policy

Every request has three parts:

  1. System instruction: role, tone, JSON-only response, no diagnosis, no therapy, no harm instructions.
  2. Negative cases: explicit refusal/redirect behavior for killing, murder, self-harm, suicide, violence instructions, and graphic harm.
  3. Student input: the latest message plus short recent context if needed.

The app also runs a small client-side classifier before calling the AI for obvious crisis or unsafe content. This is not a perfect safety system; it is a course-prototype guardrail that prevents the most obvious bad paths and makes the manual demo reliable.

7. Provider Choice

Gemini is the v1 provider.

Reasons:

  • The classroom notes mark Google Gemini API as the newer option.
  • Official docs expose direct REST text generation.
  • Official docs support structured JSON output.
  • The API can be used with plain HTTP from Android, matching the note diagram.

Alternatives remain documented:

  • OpenRouter: useful if a single OpenAI-compatible endpoint is preferred.
  • Groq: useful for fast text generation with an OpenAI-compatible API.
  • Hugging Face: useful if the team wants model/provider choice through the HF router.

See docs/adr/0001-gemini-first-ai-provider.html.

8. Safety And Support

The app is not a crisis service.

For ordinary academic worry, it gives motivational feedback. For self-harm, suicide, killing, or violence content, it returns a crisis/unsafe card instead of a normal answer.

Default Mexico resources:

  • Linea de la Vida: 800 911 2000
  • Emergencias: 911

Optional U.S. demo resource:

  • 988 Suicide & Crisis Lifeline: call, text, or chat 988

See docs/adr/0002-safety-boundary-and-negative-prompts.html.
