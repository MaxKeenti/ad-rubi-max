AnimoChat - Student Motivational AI Chat

Android implementation workspace for Programa2 in 6NM61 Programacion Movil. The app concept comes from the attached classroom notes: an Android chat where a student writes academic fears or worries and receives motivational feedback from an AI API.

Current Status

Current repo status as of 2026-06-06: the Android app builds and the Program 2 chat flow is implemented. When GEMINI_API_KEY is present in local.properties, the app uses Gemini through GeminiAiChatRepository; when the key is absent, it uses FakeAiChatRepository so the demo can still show the UI without exposing secrets.

Implemented Scope

  • Android app built with Kotlin and Jetpack Compose.
  • Chat UI for student concerns and AI motivational feedback.
  • Gemini REST provider behind a repository interface.
  • JSON response parsing before rendering in Compose.
  • Negative prompt/safety policy for killing, self-harm, suicide, and violence cases.
  • Local fake repository/fallback path for demos without an API key.
  • Manual CP2 test runbook with verification results.

Build And Verification Commands

Verification commands from the Programa2 directory:

cd Programa2
./gradlew :app:assembleDebug
./gradlew :app:testDebugUnitTest

On Windows in this workspace, the validated command used the local JDK explicitly:

$env:JAVA_HOME='C:\Users\Rubi\scoop\apps\openjdk\current'
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
.\gradlew.bat "-Dorg.gradle.java.home=C:\Users\Rubi\scoop\apps\openjdk\current" --no-daemon :app:assembleDebug --console=plain
.\gradlew.bat "-Dorg.gradle.java.home=C:\Users\Rubi\scoop\apps\openjdk\current" --no-daemon :app:testDebugUnitTest --console=plain

Both commands completed successfully on 2026-06-06. To run against Gemini, add GEMINI_API_KEY=... to local.properties; do not commit or screenshot the key.
